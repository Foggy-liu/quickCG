# 抽象概念模式（轻量引导）

> **适用对象**：抽象概念、价值观念、哲学命题。
>
> **典型例子**：自由、爱、熵、正义、美、真、善、勇气、希望

---

## ⚠️ 本模式不直接生成报告

**quick-cognition 不擅长处理抽象概念**。原因：

1. 抽象概念没有"组成结构"（自由由什么组成？没意义）
2. 抽象概念没有"时间线"（自由不会"出现 → 演化"）
3. 抽象概念的评价因人而异（什么是"真正的自由"？）

**强行用四维结构处理抽象概念会产出别扭的报告**——参考 `agent-v2-quality-review.md` 中"自由"测试的失败案例。

---

## 处理方式

检测到抽象概念后，**直接告知用户**：

```
您输入的是抽象概念，不适合 quick-cognition 的四维分析框架。

推荐方案：
1. 用 khazix-writer 写一篇深度散文（推荐）
2. 强行用 quick-cognition 处理，但产出质量可能不理想
3. 重新输入一个具体的技术/概念名称

是否切换到 khazix-writer？[Y/n]
```

---

## 如果用户选择"用 khazix-writer"（推荐路径）

**直接调用 khazix-writer**，不通过 quick-cognition 内部实现。

建议的 khazix-writer 触发语：

```
用散文的风格写一篇关于 [X] 的深度认知文章。
要求：
- 2000-3000 字
- 遵循叙事驱动、回答"为什么"、承认主观、避免 AI 味词
- 钩子开头（4 选 1：反差/悬念/类比/历史）
- 多角度拆解（哲学/历史/科学/个人等 2-4 个角度）
- 一段"核心定义"（避免"一句话定义"陷阱）
- 边界讨论（什么时候这个概念会失效）
- 回环呼应（开头钩子的 callback）
```

---

## 如果用户选择"强行用 quick-cognition"

如果用户显式要求继续（回答 "n" 并确认）：

- [ ] **不再使用四维结构**——切到散文模式
- [ ] 调用 khazix-writer 写入散文生成请求
- [ ] 散文生成结果保存为 `reports/{事物名}-essay-v{N}.md`
- [ ] HTML 包装**可选**——抽象概念认知的价值在内容，不在视觉

**不要在 quick-cognition 内部实现散文生成**——避免重复造轮子。

---

## 为什么不自己实现

- 散文需要**写作技能**（rhythm, voice, narrative arc）——这不是 quick-cognition 的专长
- 已有 khazix-writer 在做这件事
- quick-cognition 保持**单一职责**：具体知识点的结构化认知

**单一职责的好处**：
- SKILL 体积小
- 规则可执行性强
- 不会因为"什么都想做"导致质量平庸

---

## 抽象概念判断清单

**判定为抽象概念（应触发本模式）的标准**——满足以下**任一**即可：

- [ ] 无物理形态（不能触摸/看见/测量）
- [ ] 无组成结构（无法拆解为组件）
- [ ] 评价因人而异（不同人给出不同定义）
- [ ] 不随时间演变（"自由"的概念 200 年前和现在没有本质区别）
- [ ] 经常出现在哲学/价值观讨论中（不是技术讨论中）

**反例**（不属抽象概念）：
- "Agent"（具体技术，有组成结构）
- "Transformer"（具体模型，有时间线）
- "RAG"（具体方法，有评价标准）
- "创业"（虽然抽象，但是"动作"而非"概念"）

---

## 触发后的强制动作

1. [ ] 输出上面的"推荐方案"提示
2. [ ] 询问用户选择
3. [ ] 如果选 khazix-writer：给出调用建议（见上）
4. [ ] 如果用户仍坚持用 quick-cognition 处理：记录到 `reports/abstract-fallback.log`（包含：用户输入、坚持理由、最终处理方式）

**为什么记录 fallback**：方便后续优化"抽象概念判断清单"——如果发现某些输入被错误判定为抽象，可以基于 fallback 日志调整判断标准。

---

## 与其他模式的关系

| 维度 | 抽象概念模式 | 概念/技术模式 |
|------|------------|--------------|
| 是否生成报告 | ❌ 不生成 | ✓ 生成 |
| 处理方式 | 引导到 khazix-writer | 4 维结构 |
| 字数要求 | 0（不生成）| 8000+ |
| 强制规则 | 引导到合适的工具 | 4 维 + 关卡 |

**抽象概念模式是"路由器 + 引导器"，不是"生成器"**。

---

## 一句话总结

**抽象概念模式 = "承认自己不适合 + 引导到合适的工具"。** 这是 quick-cognition v2 的"诚实"——承认不擅长，然后推荐擅长的 skill，而不是硬撑。
