# 事件/现象模式

> **适用对象**：有明确起止时间的客观事件，或持续性的社会现象。
>
> **典型例子**：2008 金融危机、双 11、内卷、996、鸡娃、躺平、元宇宙、Web3 泡沫、ChatGPT 引发的 AI 浪潮

---

## 与概念/技术模式的关键区别

| 维度 | 概念/技术模式 | 事件/现象模式 |
|------|--------------|--------------|
| "历史" | 起源 → 演化的**时间线** | 事件**前因**（为什么会发生） |
| "结构" | 由**组件**组成 | 由**机制**驱动（因果链） |
| "功能" | 能**做什么** | 产生什么**影响**（后果链） |
| "本质" | 不可撼动的**核心** | 揭示了什么**规律** |

**核心变化**：把"结构/功能"维度**从"组成"语义改造为"机制/影响"语义**——事件不是"由部分组成"，是"由机制驱动"。

**为什么这样改造**：参考 `agent-v2-quality-review.md` 中"自由"和"内卷"测试——"结构维度"在事件/现象上完全不适用。

## 强制结构

| 板块 | 必启 | 字数下限 | 必标注可信度 |
|------|------|---------|------------|
| TL;DR（事件精炼描述） | ✓ | 50-100 字 | - |
| 前因（替代"历史"） | ✓ | 1500 字 | 每段 |
| 机制（替代"结构"） | ✓ | 1500 字 | 每段 |
| 影响（替代"功能"） | ✓ | 1500 字 | 每段 |
| 本质（规律揭示） | ✓ | 1000 字 | 每段 |
| 横纵交汇洞察 | ✓ | 800 字 | - |
| 回到本质 | ✓ | 500 字 | - |
| **总计** | - | **7000 字+** | - |

**字数可低于概念模式**（7000 vs 8000）——因为事件不需要"完整时间线"维度，篇幅自然减少。

## 强制规则（不遵守则重新生成）

### 前因维度（替代"历史"）

- [ ] 列出 3-5 个**直接诱因**（immediate triggers）
- [ ] 列出 2-3 个**结构性原因**（structural causes）
- [ ] **必须区分"导火索"和"火药桶"**——避免简单归因
- [ ] **不接受**"必然发生"等决定论叙述
- [ ] 可信度标注覆盖：每个诱因、每个结构性原因

### 机制维度（替代"结构"）

- [ ] **必须画出因果链图**（用 `<pre class="diagram">`）
- [ ] **必须区分触发机制和放大机制**：
  - 触发机制：什么让它**开始**
  - 放大机制：什么让它**扩散**
- [ ] 涉及多方参与者时，**每个核心方的动机都要讲**
- [ ] 可信度标注覆盖：因果链各节点

### 影响维度（替代"功能"）

- [ ] **必须区分短期影响和长期影响**（时间维度）
- [ ] **必须区分直接影响和间接影响**（因果维度）
- [ ] **必须给出可量化的影响**（如 GDP 下降 X%、失业率 Y%）
- [ ] **必须有"能力边界"小节**——什么影响被高估/被忽视
- [ ] 可信度标注覆盖：每个影响数据

### 本质维度（规律揭示）

- [ ] **必须给出 1 句话本质陈述**（用 quote-box 突出）
- [ ] **必须讲"它揭示了什么结构性规律"**——不是"事件本身的特点"
- [ ] **必须有"边界条件"小节**——这个规律在什么情况下不成立
- [ ] 可信度标注覆盖：规律归纳、边界条件

### 横纵交汇

- [ ] **必须包含与同类事件的对比**（如 2008 vs 1929、2008 vs 1997）
- [ ] **必须给出"这次和上次的本质区别"或"这次揭示的新规律"**
- [ ] 必须给出 3+ 个新判断
- [ ] 至少 1 个判断必须是"反直觉/挑战常规"的

## 必启扩展维度（强制）

**对事件/现象模式，以下扩展维度强制启用**：

| 扩展维度 | 必启 | 理由 |
|---------|------|------|
| 常见误解 | ✓ 强制 | 事件/现象有大量媒体误读，不澄清等于没讲清楚 |
| 同类对比 | ✓ 强制 | 不做对比无法精准定位"这次"的特点 |
| 类比解释 | 选启 | 抽象事件需要类比帮助理解 |

### 常见误解维度强制要求

- [ ] 至少列出 3 个**最常见的错误理解**
- [ ] 每个误解必须有"为什么错 + 正确是什么"
- [ ] 不要把"复杂"包装成"误解"——只列真正的误解
- [ ] 至少 1 个误解必须是"被高估的"（不是被低估的）

### 同类对比维度强制要求

- [ ] 至少对比 1 个同类事件
- [ ] 必须给出"差异点"和"共同点"
- [ ] 必须有"这次相对上次的本质区别"
- [ ] 对比维度可参考"前因/机制/影响"四列对比

## 禁用清单

### AI 味词（同概念/技术模式）

### 事件/现象特有禁止

- `必然发生` `不可避免` `命中注定`（决定论叙述）
- `如果当时...就好了` `要是...就不会...`（事后诸葛亮）
- `复杂归因`——必须给出可识别的诱因，不允许"原因很复杂"打太极
- 单一归因（不允许"全因为 X"）
- 阴谋论叙述（"这一切都是 X 设计的"）

---

## 写作原则

| 原则 | 说明 |
|------|------|
| **结构化归因** | 必须区分诱因和结构性原因，不允许"原因很复杂"打太极 |
| **量化影响** | 用数据替代"影响深远"等概括 |
| **对比定位** | 必须通过与同类事件对比来定位"这次"的特点 |
| **多视角** | 涉及多方参与者时，每个核心方动机都要讲 |
| **避免决定论** | 不写"必然发生""命中注定" |

## 与概念/技术模式的取舍

**什么时候用事件/现象模式 vs 概念/技术模式？**

- 如果主体是**永恒对象**（技术/概念/工具）→ 概念/技术模式
- 如果主体是**有时间维度的事件/现象** → 事件/现象模式
- 如果主体是**社会现象**（内卷/996） → 事件/现象模式
- **不要试图"用概念模式分析事件"**——结果一定是别扭的

---

## 一句话总结

**事件/现象模式 = "前因/机制/影响/本质" + 常见误解强制 + 同类对比强制。** 不允许"复杂归因"打太极——必须给出可识别的诱因链。
