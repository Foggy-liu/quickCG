# 人物/公司模式

> **适用对象**：个人、组织、公司。
>
> **典型例子**：巴菲特、乔布斯、王阳明、埃隆·马斯克、华为、特斯拉、OpenAI、Anthropic、阿里、字节

---

## 与概念/技术模式的关键区别

| 维度 | 概念/技术模式 | 人物/公司模式 |
|------|--------------|--------------|
| "历史" | 起源 → 演化的**时间线** | 个人/组织的**关键轨迹** |
| "结构" | 由**组件**组成 | 由**思想体系/业务体系**组成 |
| "功能" | 能**做什么** | 对世界产生了什么**影响** |
| "本质" | 不可撼动的**核心** | 核心**特质/哲学/价值观** |

**核心变化**：把维度语义从"组成"改造成"特质"——人物/公司不是"由部分组成"，是"由核心特质驱动"。

---

## 强制结构

| 板块 | 必启 | 字数下限 | 必标注可信度 |
|------|------|---------|------------|
| TL;DR（人物/公司画像） | ✓ | 50-100 字 | - |
| 轨迹（替代"历史"） | ✓ | 1500 字 | 每段 |
| 体系（替代"结构"） | ✓ | 1500 字 | 每段 |

| **总计** | - | **7000 字+** | - |

## 强制规则（不遵守则重新生成）

### 轨迹维度（替代"历史"）

- [ ] **按关键阶段划分**（不要纯年表）
- [ ] **每个阶段必须讲"转折点"**（改变了什么）
- [ ] **至少有 1 个反直觉转折**（不是常规的"努力成功"叙事）
- [ ] **必须区分"个人努力"和"时代红利"**——避免英雄叙事
- [ ] 至少有 1 个"被忽视的失败"或"被高估的成功"
- [ ] 可信度标注覆盖：关键节点、转折原因

### 体系维度（替代"结构"）

**对人物**：
- [ ] 必须分思想体系的：**核心主张 + 演化阶段 + 关键著作**
- [ ] 至少有 1 个"被误解的主张"（公众误读但本人其实说过不同的话）

**对公司**：
- [ ] 必须分业务的：**核心产品 + 商业模式 + 组织架构**
- [ ] 至少有 1 个"被忽视的业务线"或"被高估的核心业务"

**共同**：
- [ ] 体系不是"成就清单"，是"**核心要素的内在逻辑**"
- [ ] 可信度标注覆盖：体系各要素、要素关系

### 影响维度（替代"功能"）

- [ ] **必须区分直接影响和间接影响**（启发/带动了什么）
- [ ] **必须有负面影响的诚实讨论**（避免"伟人传"）
- [ ] **必须给出可量化的影响**（如：巴菲特 60 年复合收益率 X%）
- [ ] 必须有"能力边界"小节——这个人物/公司的影响在哪些领域被高估
- [ ] 可信度标注覆盖：影响数据、争议性影响

### 本质维度（核心特质）

- [ ] **必须给出 1 句话核心特质**（用 quote-box 突出）
- [ ] **必须有"特质 vs 现象"区分**——**特质是底层，现象是表面**
- [ ] **必须有"边界条件"小节**——这个特质在什么情况下不成立
- [ ] 必须有"被高估的特质"或"被忽视的特质"（至少 1 个）
- [ ] 可信度标注覆盖：特质归纳、边界条件

### 横纵交汇

- [ ] **必须包含与同类人物/公司的对比**（如：巴菲特 vs 芒格、华为 vs 中兴）
- [ ] **必须给出"这个人/公司的不可复制性"**——什么因素是时代/个人/运气的偶然
- [ ] 必须给出 3+ 个新判断
- [ ] 至少 1 个判断必须是"反直觉/挑战常规"的

## 必启扩展维度（强制）

| 扩展维度 | 必启 | 理由 |
|---------|------|------|
| 优缺点 | ✓ 强制 | 人物/公司必然有争议性，不写等于神化 |
| 常见误解 | ✓ 强制 | 公众认知有大量简化 |
| 同类对比 | 选启 | 选 1 个对标人物/公司 |

### 优缺点维度强制要求

- [ ] 必须给出"被低估的优点"**或**"被高估的优点"
- [ ] **必须给出"被忽视的缺点"**——不能只讲"成就有争议"
- [ ] 避免"伟人叙事"——必须有人性化讨论
- [ ] 至少 1 个"反英雄"的诚实讨论

### 常见误解维度强制要求

- [ ] 至少列出 3 个**最常见的简化/神化**
- [ ] 每个误解必须有"为什么错 + 正确是什么"
- [ ] 必须包含"被忽视的早期失败"或"被忽视的局限"

## 禁用清单

### AI 味词（同概念/技术模式）

### 人物/公司特有禁止

- **英雄叙事**（"X 之所以成功是因为...努力/天赋/远见"）
- **圣人化**（"X 是伟大的/卓越的/非凡的"）
- **性格决定论**（"X 性格坚毅所以...成功"）
- **时代无关论**（把成就全归功于个人，忽视时代红利）
- **伟人传式语言**（"X 改变了世界"——除非有具体证据）

---

## 写作原则

| 原则 | 说明 |
|------|------|
| **反英雄叙事** | 必须有人性化讨论，避免神化 |
| **时代归因** | 必须区分个人努力和时代红利 |
| **量化影响** | 用数据替代"影响深远"等概括 |
| **诚实局限** | 必谈缺点和被忽视的失败 |
| **特质 vs 现象** | 特质是底层，现象是表面——必须区分 |

## 与概念/技术模式的取舍

**什么时候用人物/公司模式 vs 概念/技术模式？**

- 如果主体是**技术/概念/方法/工具** → 概念/技术模式
- 如果主体是**人或组织** → 人物/公司模式
- **判断依据**：能否画出"思想/业务体系"——能就是人物/公司模式

---

## 一句话总结

**人物/公司模式 = "轨迹/体系/影响/本质" + 强制诚实讨论优缺点 + 强制区分特质与现象。** 避免一切"伟人传"叙事——人物/公司报告的核心价值是"反神化"。
